## What
Ship the symmetric half of the bridge in this package (spec §9, §10 — decided 2026-07-14: ship here so AgentViewKit consumes any ACP agent uniformly), in `Sources/FoundationModelsACP/Bridge/`:

- A client-side utility that folds an `AsyncStream<SessionUpdate>` into a growing FoundationModels `Transcript`:
  - `agent_message_chunk` → `.response` text
  - `tool_call` / `tool_call_update` → `.toolCalls` / `.toolOutput` (correlated by `toolCallId`)
  - `agent_thought_chunk`, `plan`, and permission requests → custom `.structure` segments
- The ACP client thereby becomes just another producer of the same Transcript AgentViewKit already renders — the inverse of the §7 mapping.
- Round-trip property: a turn mapped FM → ACP (bridge task) and back through this utility reproduces an equivalent Transcript (lossless round-trip, spec §7).
- Apply the same straggler policy as the session-update stream task: late `tool_call_update`s still fold in.
